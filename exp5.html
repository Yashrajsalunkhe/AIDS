Experiment 5

Program to Demonstrate Concept of DTD and Its Types

Objective

To provide the fundamentals of a valid XML document.

Theory

Introduction to XML

XML (Extensible Markup Language) is a text-based markup language derived from SGML. It is used to store and organize data rather than presenting it like HTML.

  • XML is extensible: You can create your own tags.
  • XML carries data, not presentation: It stores data independently of how it is displayed.
  • XML is a public standard: Developed by W3C, it is an open standard.

XML Example

<?xml version="1.0"?>
<contact-info>
    <name>Tanmay Patil</name>
    <company>TutorialsPoint</company>
    <phone>(011) 123-4567</phone>
</contact-info>

XML Validation

XML validation ensures that an XML document conforms to a specific structure using a Document Type Declaration (DTD).

Well-formed XML

An XML document is well-formed if it adheres to syntax rules, such as:

  • Each opening tag has a closing tag.
  • Tags are properly nested.
  • Only one root element exists.

Valid XML

If an XML document has an associated DTD and conforms to it, it is considered valid.

Internal DTD

<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<!DOCTYPE address [
    <!ELEMENT address (name,company,phone)>
    <!ELEMENT name (#PCDATA)>
    <!ELEMENT company (#PCDATA)>
    <!ELEMENT phone (#PCDATA)>
]>
<address>
    <name>Tanmay Patil</name>
    <company>TutorialsPoint</company>
    <phone>(011) 123-4567</phone>
</address>

External DTD

External DTD is referenced using a SYSTEM attribute.

<?xml version="1.0" encoding="UTF-8" standalone="no"?>
<!DOCTYPE address SYSTEM "address.dtd">
<address>
    <name>Tanmay Patil</name>
    <company>TutorialsPoint</company>
    <phone>(011) 123-4567</phone>
</address>

Conclusion

We have studied XML documents and their validation types using DTD.
